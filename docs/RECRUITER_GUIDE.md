# Recruiter guide

This repository is designed to be read in less than ten minutes by a recruiter or technical screener.

## One-line signal

Junior Data / DataOps / MLOps / AI engineering literacy for Swiss employers across regulated, data-intensive and scale-engineering environments.

## The core proof

The core proof is not the long list of languages. The core proof is:

```text
Python + SQL + data quality + PostgreSQL + JSON/JSONL + Docker + CI/CD + monitoring + MLflow + governance documentation
```

## Employers this repository should speak to

| Employer family | What they should see |
|---|---|
| Banks / fintech / financial infrastructure | SQL, auditability, controls, data quality, monitoring, MLOps, AI governance |
| Insurance | claims / risk data quality, reproducible pipelines, privacy controls, explainability |
| Pharma / life sciences | traceability, validation discipline, controlled documentation, data lineage |
| Retail / e-commerce | event data, APIs, analytics, model monitoring, product/customer data formats |
| Industry / manufacturing | sensor/event data, anomaly monitoring, batch/stream concepts, reliability |
| Logistics / transport / energy | operational data pipelines, latency, alerts, incident runbooks |
| Consulting / integrators | migration, reconciliation, documentation, multi-stack literacy |
| Cloud / platform / big tech | CI/CD, containers, Kubernetes, Terraform, observability, scalable data systems |
| Public sector / research | documentation, reproducibility, governance, transparent data workflows |

## What this repo does not claim

It does not claim senior-level proficiency in every language. It shows practical orientation: what each technology is, where it fits, and why it matters in Data & AI engineering.
