# Swiss recruitment positioning

## Target

Data / DataOps / MLOps / AI Engineer for Swiss regulated, data-intensive and scale-engineering employers.

## Market-compatible roles

- Data Engineer
- DataOps Engineer
- IT Production Engineer
- Application & Data Support Engineer
- Risk & Compliance Data Analyst
- Analytics Engineer
- MLOps Engineer
- AI Platform Engineer
- Data Platform Engineer
- Data Quality Analyst
- Technical Data Analyst

## Core message

I build reliable data and ML systems: ingestion, SQL, data quality, monitoring, MLflow, Docker, CI/CD, audit documentation and AI governance.

## Why this matters in Switzerland

The Swiss market is not one single employer type. The same core engineering signal can be read by banks, insurers, pharma/life-sciences companies, retailers, industrial companies, logistics/energy firms, consulting firms, public institutions, big-tech teams and scale-ups.

The transferable priorities are:

```text
reliable data
clear SQL and Python foundations
production monitoring
documentation discipline
data privacy awareness
API and file-format literacy
CI/CD and container literacy
model governance and auditability
```

## Sector translation

| Sector | Translation of the same project |
|---|---|
| Banking / finance | transaction monitoring, risk/compliance analytics, audit trail, model governance |
| Insurance | claims quality, fraud triage, pricing data, privacy controls |
| Pharma / health / life sciences | validated data flows, traceability, reproducibility, controlled documentation |
| Retail / e-commerce | customer events, product data, recommendation monitoring, experimentation data |
| Industry / manufacturing | sensor data, anomaly detection, quality monitoring, operational alerts |
| Logistics / transport / energy | route / operational events, reliability monitoring, incident response |
| Consulting / system integrators | data migration, reconciliation, modernization, multi-client documentation |
| Cloud / platform / big tech | scalable data systems, CI/CD, Kubernetes, observability, ML platform foundations |
| Public / academic / research | reproducible workflows, transparent documentation, governed data pipelines |

## Recommended public GitHub boundary

- `jedha-rncp35288-portfolio`: certification evidence only.
- `swiss-data-ai-engineering-lab`: personal technical literacy lab for Swiss Data & AI recruitment.
- `IchiGrid-EA`: separate personal project, not mixed with professional Data & AI recruitment positioning.
